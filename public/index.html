Agree-Dis Service Api

Agree-dis is a Service that Manage a Debation between 2 Users
- 2 users publish an idea ,people can vote if they agree or not.

This is a documentation for how to use this Service, this documentation contains requests, responses and errors

database Global methods Debates methods
Debates Structure

{
basic_info:{
   title:String,
   img:String,
   voters:[String]
   status:Number
/status can be 0=waiting,1=rejected,2=running,3=closed
},
owner:{
   owner_id:Number,
   owner_votes:Number
},
collaborator:{
   collaborator_id:Number,
   collaborator_votes:Number
 }
}

Users Structure

{
id:Number,
profile:{
email:String,
password:String,
name:{
   first:String,
   last:String
},
age:Number,
imgSrc:String,
},
debates:[String],/*will hold the debates id*/
claps:Number,
vip:Number,
notifications:[String]

Sign-in
'POST' Request
https://agree-dis.herokuapp.com/profile/signIn

Parametrs: { email },{ password } you can't access any of the service route without Signin

Back to Top

on Success: {Success: you are signed in as { first: ' ', last: ' ' } }
Sign-out
'GET' Request
https://agree-dis.herokuapp.com/profile/signIn

Sign out will Destroy the User Session

Get Debate By Debate ID
'GET' Request
https://agree-dis.herokuapp.com/debates/:debate_id

Visitors can get only the public Debates.
debate owner can Search for any kind of debate.

Back to Top
on Error: {Error: Not authrized} - this Error means that you need to sign in first
on Error: {Error: no Debate Found} - Wrong debate id
on Success: Debate Document
